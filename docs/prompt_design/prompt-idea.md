# 営業相談チャットのプロンプトデザインについて

営業相談チャットでは、相談者に対して、丁寧かつ敬意のある言葉で、営業の手助けになるようなコミュニケーションを行っていきます。
これのプロンプトおよびメソッドをデザインしてください。

# 現状
Message.for_openai(messages) メソッドについて、プロンプトを生成している。これをリプレイスしたい。

# 取得可能な情報

1. 製品RAGを通じて、製品情報データを取得可能　※製品ごとに1つ
2. 先輩営業RAGを通じて、先輩の過去の営業トークデータを取得可能　※製品ごとに複数の先輩営業RAGがある。

# プロンプトの方向性について

精度と速度のバランスを考えて、下記のイメージ。

1. 製品情報が必要のない一般的な内容であれば、RAGに問い合わせることの無いプロンプト
2. 製品情報が必要な問い合わせであれば、製品情報RAGから製品情報データを取得し、プロンプトに入れ込む
3. 先輩営業への相談であれば、先輩営業RAGを通じて、先輩の過去の営業トークデータし、プロンプトに入れ込む　※先輩営業が指定されていない場合は、相談相手を指定するように返す
4. 製品情報と先輩営業の過去データの両方が必要であれば、 両方のRAGに問い合わせた上で、プロンプトに入れ込む

## 考えられるインプット

- product_id　: 製品ID　必須
- sales_expert_id : 先輩営業ID　任意
- messages : logs of current chat　必須
- query : user input 必須
